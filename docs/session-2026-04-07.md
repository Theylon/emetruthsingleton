## Google Calendar Integration

### How it works
Clicking "Schedule a call" opens a blurred overlay modal containing the Google Calendar scheduling page in an iframe.

- No dependency on `calendar.schedulingButton` script (was unreliable â€” async timing, hidden element `.click()` issues)
- URL: `https://calendar.google.com/calendar/appointments/schedules/AcZssZ29Bm-_F3Vq03bG4OG5Vz6oK5hvspwO0DNu6JRynAdlPtSyIKv9L-2DD0cyTpkKVEOrRNs15RWL?gv=true`
- Closes on click-outside or Escape key
- On mobile: slides up from bottom (90vh sheet)
- Google's scheduling page has no `X-Frame-Options`, so iframe embeds without issue

### What was tried first (and why it failed)
1. `calendar.schedulingButton.load()` on a hidden off-screen element â†’ unreliable async timing
2. Google Calendar CSS link (`scheduling-button-script.css`) was injecting global styles that overrode our button design â†’ removed CSS, kept JS only
3. Zero-size wrapper clipping â†’ still unreliable for `.click()` triggering
4. Final solution: own iframe modal, no external script dependency

## Mobile Optimisations

The page was originally `height: 100dvh; overflow: hidden` with no mobile override â€” all content below the fold was clipped.

### Fixes
- `.singleton-page` unlocked on mobile: `height: auto; overflow-y: auto`
- `.singleton-shell` and `.singleton-panel` set to `height: auto; min-height: unset`

### Polish
- Title: `clamp(2.6rem, 12vw, 3.6rem)` â€” was `19vw`, too aggressive
- Cards: 2-column grid (third card spans full width) instead of 3 stacked
- Cards: gold left-border accent on mobile (replaces hover which doesn't work on touch)
- Card hover transforms disabled on mobile
- Action panel: `border-radius: 1rem`, tighter padding
- Orbs: scaled to `vw` units so they frame rather than overflow
- Mobile background: soft gold/dark radial gradient (3D canvas is hidden on mobile)
- 3D canvas: fades in (`opacity 0â†’1`) once both GLB models finish loading
